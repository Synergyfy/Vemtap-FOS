# 17. CALCULATION ENGINE DESIGN (LOGIC LAYER)

This defines how raw data becomes financial intelligence. All calculations should be implemented as services/functions that read from the database and output metrics used across the dashboard.

---

## A. CORE DEFINITIONS (SOURCE OF TRUTH)

- Gross Revenue = SUM(transactions.amount WHERE type IN ['subscription','sms','enterprise'])
- Total Costs = SUM(transactions.cost) + SUM(expenses.amount) + SUM(commission payouts)
- Net Profit = Gross Revenue - Total Costs
- MRR (Monthly Recurring Revenue) = SUM(monthly_payment FROM active businesses)

---

## B. COMMISSION CALCULATION

For each transaction (subscription):

- commission = amount \* commission_rate (e.g. 30%)
- Store in transactions.profit = amount - commission - cost

Agent Earnings:

- agent_total_commission = SUM(commission WHERE agent_id = X)

---

## C. SMS PROFIT ENGINE

Per record in sms_usage:

- total_cost = sms_count \* cost_per_sms
- total_revenue = sms_count \* selling_price_per_sms
- total_profit = total_revenue - total_cost

Aggregate:

- sms_total_profit = SUM(total_profit)

---

## D. DAILY / WEEKLY / MONTHLY AGGREGATIONS

For any period (start_date → end_date):

- revenue = SUM(transactions.amount)
- profit = SUM(transactions.profit)
- new_businesses = COUNT(businesses WHERE join_date in range)

These power all charts and reports.

---

## E. CHURN CALCULATION

- churn_rate = (lost_customers / total_customers_start_period) \* 100

Where:

- lost_customers = businesses that moved from active → inactive in period

---

## F. CONVERSION ENGINE (QRTHRIVE → VEMTAP)

- conversion_rate = (converted_to_vemtap / qr_users) \* 100

- revenue_from_conversions = SUM(monthly_payment of converted users)

---

## G. BUDGET VS ACTUAL ENGINE

For each budget:

- actual_revenue = SUM(transactions.amount within budget period)
- performance_ratio = actual_revenue / target_revenue

Output:

- If >= 1 → On Track
- If 0.7–0.99 → At Risk
- If < 0.7 → Off Track

---

## H. TARGET BREAKDOWN ENGINE

Given:

- monthly_target_revenue

Compute:

- daily_target = monthly_target / 30
- weekly_target = monthly_target / 4

Required businesses:

- required_businesses = target_revenue / ARPU

Agent requirement:

- businesses_per_agent = required_businesses / total_active_agents

---

## I. FORECAST ENGINE

For each future period:

- projected_businesses = current_businesses \* (1 + growth_rate - churn_rate)
- projected_revenue = projected_businesses \* ARPU
- projected_profit = projected_revenue \* (1 - commission_rate) - fixed_costs

Run this iteratively for 3, 6, 12 months.

---

## J. CASH FLOW ENGINE

For a given period:

- inflow = SUM(cash_flow.amount WHERE type = 'inflow')
- outflow = SUM(cash_flow.amount WHERE type = 'outflow')

- net_cash = inflow - outflow

Runway:

- runway_months = current_cash_balance / monthly_burn

---

## K. BREAK-EVEN ENGINE

- break_even_revenue = fixed_costs / (1 - commission_rate)

- gap_to_break_even = break_even_revenue - current_revenue

---

## L. SCENARIO SIMULATION ENGINE

Inputs:

- growth_rate
- churn_rate
- commission_rate
- pricing (ARPU)

Run forecast engine with different inputs:

- Best Case (high growth, low churn)
- Expected Case
- Worst Case (low growth, high churn)

Output:

- Profit comparison table
- Risk indicators

---

## M. ALERT & DECISION ENGINE

Trigger rules:

- If churn_rate > 10% → Alert
- If profit < 0 → Alert
- If revenue growth < target → Alert

Decision outputs:

- "Increase agent acquisition"
- "Push plan upgrades"
- "Reduce cost"

---

## N. SNAPSHOT GENERATION (PERFORMANCE)

Daily cron job:

- Calculate all key metrics
- Store in metrics_snapshot table

This ensures fast dashboard loading and historical tracking.

---

## DESIGN PRINCIPLES

- All calculations must be reproducible from raw data
- Avoid hardcoding values (use assumptions/config table)
- Separate calculation logic from UI
- Use scheduled jobs for heavy computations

---

This calculation engine transforms your database into a decision-making system. Without it, the dashboard is just visuals. With it, the system becomes your financial brain.
